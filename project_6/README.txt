#Assignment 6
gcc (Ubuntu 9.3.0-17ubuntu1~20.04) 9.3.0

Usage:
    assign_6 -r <pcap_filename>
    assign_6 -h
Options
    -r Packet capture filename (e.g. test.pcap)
    -h Display help message

Η επεξεργασία των δεδομένων του αρχείου .pcap γίνεται μέσω των συναρτήσεων της βιβλιοθήκης libpcap. Συσγκεκριμένα, 
μέσω των συναρτήσεων pcap_open_offline(pcap_file, err) και την pcap_loop().

Για την μέτρηση των network_flows που δημιουργήθηκαν κατά τη σύλληψη πακέτων, δημιουργήθηκε ένα struct με τα 5
πεδία που καθορίζουν το flow σύμφωνα με τον ορισμό του (Source IP and Port, Destination IP and Port and Transfer 
Layer Protocol - TCP or UDP), και μία συνδεμένη λίστα κόμβων στην οποία αποθηκεύονταν μοναδικά οι κόμβοι, δηλαδή
δεν υπάρχουν διπλότυπα.

Για τα ερωτήματα 9, 10 ισχύουν τα παρακάτω:
    9) Για να καταλάβουμε αν ένα πακέτο TCP έχει επαναμεταδωθεί πρέπει να ισχύουν τα εξής:
        - Όταν το πακέτο δεν είναι kept-alive
        - Όταν το payload έχει μέγεθος μεγαλύτερο του 0, ή όταν τα flags SYN ή FIN έχουν τιμή 1
        - Όταν ο αναμενόμενος αριθμός ακολουθίας είναι μεγαλύτερος από τον αριθμό ακολουθίας του πακέτου.
       Για να ελέγξουμε τα παραπάνω, δημιουργήθηκε ένα struct με πεδία το network_flow στο οποίο ανοίκει το κάθε TCP
       packet, το tcp header του κάθε πακέτου και το payload του. Κάθε TCP πακέτο αποθηκεύεται αντίστροφα σε μία λίστα 
       (ώστε να ελαχιστοποιηθεί στατιστικά ο αριθμός των αναδρομικών κλήσεων για την αναζήτηση μέσα σ' αυτή) και για
       κάθε μετάδοση πακέτου ελέγχουμε αν ισχύουν οι παραπάνω συνθήκες συγκρίνοντας τα πεδία του TCP πακέτου που
       εξετάζουμε και των υπολοίπων που έχουν μπει μέσα στη λίστα. Με αυτή την τεχνική βέβαια δεν εξετάζεται η περίπτωση
       TCP Fast Retransmission και TCP Spurious Retransmission. Παρατηρώντας το αποτέλεσμα, δεν θεωερείται σωστό, καθώς
       σημαδεύονται ως reatransmitted περισσότερα πακέτα απ' όσα θα έπρεπε.

    10) Το UDP δεν προβλέπει αναμετάδοση πακέτων, οπότε είναι αδύνατη η ανίχνευση retransmitted UDP πακέτου.
